01

Visão estratégica

O BRIF é uma plataforma SaaS multi-tenant com arquitetura white label managed, licenciada a agências de publicidade de pequeno e médio porte. Cada agência opera em ambiente isolado — com identidade visual própria configurada pela equipe BRIF — enquanto compartilha infraestrutura e inteligência artificial da plataforma.

Proposta de valor: Um único sistema white label que transcreve, consolida, organiza e gera documentos com IA — entregando à agência a identidade de um produto próprio, sem o custo e a complexidade de desenvolvê-lo do zero.

Problema central

  • Briefings perdidos em conversas de WhatsApp e e-mails não rastreáveis
  • Retrabalho por decisões de reunião que nunca foram registradas formalmente
  • Clientes sem visibilidade real do andamento dos projetos
  • Propostas criadas do zero em ferramentas externas a cada novo projeto
  • Contratos e fornecedores gerenciados de forma manual e descentralizada
  • Cada agência usa 5–8 apps desconectados para um único projeto

Arquitetura multi-tenant white label

Modelo de isolamento

Cada agência cliente é um tenant com schema PostgreSQL próprio. O isolamento é físico — nenhuma query de um tenant acessa dados de outro.

IsolamentoSchema-per-tenant no PostgreSQL — isolamento físico completo
Subdomínio padrãoagencia.brif.app — provisionado pela equipe BRIF
Domínio customizadoapp.minhaagencia.com.br — CNAME + SSL automático via Cloudflare
Identidade visualLogo (3 variações), paleta de cores, fontes — configurados no onboarding
Templates de propostaCriados com a marca da agência no onboarding pela equipe BRIF
E-mail transacionalDisparos com nome, logo e domínio da agência via Resend subdomain auth

Onboarding 100% managed

Decisão de produto: o setup de cada tenant é feito exclusivamente pela equipe BRIF. A agência não altera logo, cores, domínio ou templates após o onboarding. SLA: até 5 dias úteis após assinatura do contrato.

05

Módulos e funcionalidades

M1 MVP
Briefing e transcrição IA
Upload de áudio/vídeo ou integração Meet/Zoom. Whisper large-v3 com diarização. Claude gera briefing estruturado. Fluxo de aprovação 3 etapas com notificações white label.
M2 MVP
Gestão de projetos e tarefas
Kanban e lista. Fases configuráveis. Subtarefas com checklists, prazos e responsáveis. Upload de arquivos e comentários versionados. Menção com @nome.
M3 MVP
Calendário e roadmap
Calendário unificado com integração Google Calendar. Alertas automáticos 72h, 24h e no dia. Gantt interativo com dependências no pós-MVP.
M4 MVP
Portal do cliente
Status em tempo real, aprovação digital de briefing, acesso a propostas. Magic Link por projeto — sem senha. Interface responsiva 100% com a marca da agência.
M5 MVP
Propostas visuais com IA
Claude transforma briefing em proposta HTML com logo e cores da agência. Templates por tipo de campanha. Gerente decide quando publicar. Versionamento completo.
M6 MVP
Debriefing e ajustes
Mesmo fluxo de transcrição aplicado ao debriefing. IA identifica ajustes vs. proposta anterior (diff). Nova versão com marcação visual das mudanças.
M7 MVP
Produção e contratos
Upload de contratos com metadados. Aprovação dupla obrigatória: Jurídico + Financeiro. Disparo automático de e-mail para fornecedores. Status de rastreio.
M8 MVP
Gestão de documentos
Repositório central por projeto com versionamento. Categorias: briefing, criação, produção, financeiro, legal. Controle de acesso por perfil e documento.
M9 MVP
Painel admin (admin.brif.app)
Ferramenta interna exclusiva da equipe BRIF. Criação e configuração completa de tenants: white label, domínio, templates, e-mail, usuários. Dashboard global: tenants, MRR, churn, custo de IA por tenant em tempo real. Impersonation segura com log de auditoria completo.

Stack técnica

Frontend
Next.js 14 + Tailwind + shadcn/ui
Backend
Node.js + tRPC
Banco
PostgreSQL + Prisma (schema-per-tenant)
Auth
NextAuth.js + JWT + Magic Link
Transcrição IA
OpenAI Whisper API (large-v3)
IA de linguagem
Anthropic Claude API (claude-sonnet-4)
Storage
Cloudflare R2
Fila de jobs
BullMQ + Redis
E-mail
Resend + subdomain auth por tenant
Billing
Stripe + Customer Portal
DNS / SSL
Cloudflare Workers
Infra
Vercel (frontend) + Railway (backend)
Por que schema-per-tenant? Isolamento físico total — impossível vazar dados entre tenants por bug de query. Facilita backup individual, auditorias futuras (LGPD) e escala até ~500 tenants no mesmo cluster sem degradação.
08

Escopo do MVP (0–6 meses)

Estratégia: dogfooding pela própria agência do founder → beta fechado com agências parceiras já mapeadas → go-live via demo call.

Incluído no MVP

  • Multi-tenancy com schema isolation + subdomínio padrão por tenant
  • Painel Admin (admin.brif.app): onboarding managed completo, white label total, monitoramento
  • White label: logo, cores, fontes, domínio customizado, e-mail, templates de proposta
  • Billing Stripe: planos Basic e Pro com recorrência mensal
  • Todos os 8 perfis de usuário com RBAC granular
  • Upload de áudio + transcrição Whisper assíncrona (BullMQ)
  • Briefing Consolidado via Claude + fluxo de aprovação 3 etapas
  • Portal do cliente com Magic Link (status, briefing, proposta, mensagens)
  • Geração de proposta visual HTML via Claude com marca da agência
  • Kanban + lista + upload de arquivos + comentários nas tarefas
  • Calendário básico com integração Google Calendar e alertas de prazo
  • Repositório de documentos por projeto com categorias e versionamento
  • E-mail transacional white label via Resend

Pós-MVP (6–12 meses)

  • App mobile React Native (iOS + Android)
  • Módulo de debriefing com diff automático de propostas
  • Módulo de produção e contratos com fluxo de aprovação
  • Gantt interativo com dependências entre tarefas
  • Integração Google Drive / Dropbox
  • Integração DocuSign para assinatura eletrônica
  • Busca full-text em transcrições e documentos
  • Dashboard de métricas operacionais da agência
  • Self-service parcial de white label pelo Tenant Admin
  • Suporte a espanhol para expansão LATAM
09

Regras de negócio críticas

RN-01 Projetos só são criados após aprovação do briefing pelo cliente — sem exceção ou bypass
RN-02 Propostas só ficam visíveis ao cliente após publicação explícita pelo Gerente — nunca automático
RN-03 Contratos só são disparados a fornecedores após aprovação dupla: Jurídico + Financeiro
RN-04 Clientes externos nunca acessam dados internos da agência: tarefas, equipe, finanças, outros projetos
RN-05 Dados de diferentes tenants são fisicamente isolados em schemas PostgreSQL separados
RN-06 Versões aprovadas de briefing e proposta são imutáveis — toda mudança cria nova versão auditável
RN-07 Transcrição passa por revisão humana antes do briefing formal — IA auxilia, humano decide
RN-08 Configurações de white label (domínio, templates, e-mail) são gerenciadas exclusivamente pela equipe BRIF
RN-09 Impersonation pelo Admin é sempre logada com operador, motivo e timestamp completo
RN-10 E-mails transacionais saem 100% com a marca da agência — BRIF nunca aparece ao cliente final
RN-11 Tenants que excedem o limite de plano recebem aviso automático; bloqueio após 7 dias sem upgrade
RN-12 Custos de IA por tenant são monitorados em tempo real no painel admin para controle de margem
